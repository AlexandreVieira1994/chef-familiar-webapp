# Logbook do projeto

Este diretorio guarda a memoria operacional do projeto Chef Familiar.

Usar estes ficheiros no inicio de cada sessao:

1. `codex-handoff.md` - estado herdado, contexto historico e validacoes anteriores.
2. `codex-rules.md` - regras permanentes para o Codex ao trabalhar neste repo.
3. `planning.md` - prioridades, plano de trabalho e fila de decisoes.
4. `project-log.md` - registo cronologico das sessoes e alteracoes.

Objetivo: reduzir contexto perdido entre sessoes, tornar as decisoes rastreaveis e garantir que commit, push, deploy e validacao em producao nao ficam esquecidos.
